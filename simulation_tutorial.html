1数据生成：我们要模拟的数据流形是什么？reward 是什么？

这一节对应 train.py 和 test_func.py。在做扩散之前，论文先人为构造一个「干净」的数据分布和一个奖励函数，这样才能定量验证「引导有没有把样本推向高奖励、同时有没有偏离数据流形」。

1.1 两种数据流形

(A) linear_latent —— 线性低维子空间（默认实验）

直觉 想象 64 维空间里，真实数据其实只活在一个 16 维的「平面」上（一个线性子空间）。这模拟了真实世界的「数据有低维结构」——比如所有合法的图片只占像素空间极小一块。
train.py · LinearLatentData.generate_x (L136) 
def generate_x(self, N):
    # A 是 (d_inner=16, d_outer=64) 的行正交矩阵（来自 ortho_group）
    # x = 16维标准高斯 @ A  →  落在 A 的行空间里，是 R^64 中的一个 16 维子空间
    x = np.random.randn(N, self.d_inner).dot(self.A)
    return x

子空间 \(\mathcal{A}\) 由 ortho_group.rvs(64)[:16,:] 取前 16 行得到（正交基）。「在流形上」就是指 \(x\) 落在这个 16 维子空间里。

论文关心一个量——off-support ratio（偏离比）：把任意点 \(x\) 分解成「平行于子空间的分量 \(x_\parallel\)」和「垂直分量 \(x_\perp\)」，比值 \(r = \|x_\perp\| / \|x_\parallel\|\)。真实数据 \(r=0\)；引导如果把样本推出子空间，\(r\) 就变大。

train.py · off_support_ratio (L110) 
parallel = samples.dot(self.A.T).dot(self.A)   # 投影到子空间 = x A^T A
vertical = samples - parallel                   # 垂直分量
ratio = norm(vertical) / (norm(parallel)+1e-8)  # r = ||x_perp|| / ||x_para||

(B) unit_ball —— 非线性流形（单位球面）

train.py · UnitBallData.generate_x (L184) 
x = np.random.multivariate_normal(mean=0, cov=I, size=N)   # 64 维高斯
x = x / np.linalg.norm(x, axis=-1, keepdims=True)          # 归一化到单位球面 ||x||=1

这里数据流形是 \(\{x : \|x\|=1\}\)，一个弯曲（非线性）的流形。off-support ratio 定义成 \(\|x\|-1\)（偏离球面多远）。论文用它来说明：即使流形是非线性的，G_loss 依然能贴着流形走。

优化时每一步要拿到当前样本处奖励的梯度 \(g=\nabla f(x)\)。每个样本各算各的（并行优化一整批）：

2加噪（forward process）：处理之后我们得到了什么样的 data？

直觉 扩散模型的核心套路：正向把数据一点点泡进噪声里，直到变成纯高斯噪声（这一步没有学习，是固定的数学公式）；反向训练一个网络把噪声一点点擦掉。加噪过程存在的唯一目的，是定义一条「数据 ⇄ 噪声」的可逆通路，让神经网络有东西可学。

2.1 加噪的数学

定义一串噪声强度 \(\beta_1,\dots,\beta_T\)（schedule），令 \(\alpha_t = 1-\beta_t\)，累积乘积 \(\bar\alpha_t = \prod_{s\le t}\alpha_s\)。则从干净数据 \(x_0\) 一步直达任意 \(t\) 时刻的噪声样本：

$$ x_t = \sqrt{\bar\alpha_t}\, x_0 + \sqrt{1-\bar\alpha_t}\,\varepsilon, \qquad \varepsilon\sim\mathcal{N}(0,I) $$

等价地：\(q(x_t\mid x_0)=\mathcal{N}\!\big(x_t;\ \sqrt{\bar\alpha_t}x_0,\ (1-\bar\alpha_t)I\big)\)。

  • \(t\) 小（接近 0）：\(\bar\alpha_t\approx 1\) → \(x_t\approx x_0\)，几乎没噪声。
  • \(t\) 大（接近 T）：\(\bar\alpha_t\approx 0\) → \(x_t\approx\varepsilon\)，几乎是纯噪声。

2.3 加噪在代码里只有一行

diffusion.py · p_losses (L109) 
def p_losses(self, x_start, t, classes=None):
    noise = torch.randn_like(x_start)                                       # ε ~ N(0,I)
    x = extract(self.sqrt_alphas_cumprod, t, x_start.shape) * x_start \
        + extract(self.sqrt_one_minus_alphas_cumprod, t, x_start.shape) * noise   # x_t = √ᾱ_t·x_0 + √(1-ᾱ_t)·ε
    return F.mse_loss(self.model(x, t, classes), noise)
处理之后我们得到了什么？ 我们并没有得到「一份新数据集」，而是得到了一个带噪样本的生成规则：给定任意干净样本 \(x_0\) 和任意时间步 \(t\)，立刻能造出 \(x_t\)。训练时对每个样本随机抽一个 \(t\)（见下一节）。
几何上的关键变化：原本数据挤在 16 维子空间上（测度为 0，score 在子空间外没定义）；加了高斯噪声后，\(x_t\) 的分布胖了一圈、铺满整个 64 维空间，于是 \(\nabla\log q(x_t)\)（score）处处有定义、可被网络学习。这就是为什么必须先加噪。

3denoiser 训练：怎么让网络学会 predict noise？

3.1 训练目标

直觉 给网络看一个被噪声污染的 \(x_t\)（外加告诉它现在是第几步 \(t\)），让它猜出当初加进去的那团噪声 \(\varepsilon\)。猜得越准，说明它越懂「干净数据长什么样」。训练就是反复：抽一个真实样本 → 随机选 \(t\) → 加噪 → 让网络预测噪声 → MSE 回传。

损失（DDPM 的标准简化损失）：

$$ \mathcal{L}(\theta)=\mathbb{E}_{x_0,\,t,\,\varepsilon}\big\|\,\varepsilon_\theta(x_t,t)-\varepsilon\,\big\|^2,\qquad x_t=\sqrt{\bar\alpha_t}x_0+\sqrt{1-\bar\alpha_t}\,\varepsilon $$ diffusion.py · forward + p_losses (L116, L109) 
def forward(self, img, classes=None):
    t = torch.randint(0, self.num_timesteps, (img.shape[0],)).long()  # 每个样本随机一个时间步 t
    return self.p_losses(img, t, classes)
# p_losses 里：return F.mse_loss(self.model(x_t, t), noise)   ← 预测噪声 vs 真噪声

论文对应 · 为什么「预测噪声」= 学 score 网络学到的最优解是条件期望 \(\varepsilon_\theta(x_t,t)\to \mathbb{E}[\varepsilon\mid x_t]\)。而 score（对数密度梯度）与噪声只差一个缩放： $$ s(x_t,t):=\nabla_{x_t}\log q(x_t)= -\frac{\varepsilon_\theta(x_t,t)}{\sqrt{1-\bar\alpha_t}} $$ 代码里这个换算到处都是，比如 uncond_score = -pred_noise / sqrt_one_minus_alphas_cumprod。记住这条换算，第 5 节的 guidance 全靠它。

3.2 denoiser 网络长什么样（Unet1D）

对应 unet_1d.py。它是一个把数据当成「长度=64 的一维信号」处理的 1D U-Net。你不需要逐行读懂，知道它的输入输出接口即可：

输入含义
x (B, 64)带噪样本 \(x_t\)（内部 reshape 成 (B,1,64) 走 1D 卷积）
time (B,)时间步 \(t\)，经 SinusoidalPosEmb 编码后注入每个 ResBlock（让网络知道「现在噪声多大」）
classes条件嵌入接口（本仓库没用，见下方坑②）

输出：(B, 64)，即预测的噪声 \(\varepsilon_\theta\)。结构是标准 U-Net：下采样 → 中间块（带 attention）→ 上采样，配 skip-connection。

命名坑② —— 极其重要 main.py:371 建模型时是 Unet1D(dim=64, conditional=False)。也就是说网络根本不吃 classes 这个输入，它永远是个无条件去噪器。
那 sampling 时传进去的 classes 是什么？——是引导的目标值 \(y\)（target reward），它不进网络，而是在第 5 节的 cond_score_v1/v2 里被当作引导项的一部分使用。
结论：本仓库的「条件生成」完全靠 training-free guidance（采样时改 score）实现，没有重新训练一个条件模型。这正是 DPS 这类方法的卖点：一个无条件预训练模型 + 采样时引导，就能解各种逆问题。

4采样（无条件 reverse process）

直觉 训练好 denoiser 后，从纯噪声 \(x_T\sim\mathcal{N}(0,I)\) 出发，反复「预测噪声 → 擦掉一点 → 加一点小随机」，走 T 步回到 \(x_0\)。这就是 DDPM 采样。每一步都用到一个中间产物：从当前 \(x_t\) 预估的干净样本 \(\hat x_0\)，它是后面所有 guidance 的主角。

4.1 Tweedie 公式：从 \(x_t\) 预估 \(\hat x_0\)

把第 3 节加噪公式 \(x_t=\sqrt{\bar\alpha_t}x_0+\sqrt{1-\bar\alpha_t}\varepsilon\) 反解，并用网络预测的 \(\varepsilon_\theta\) 代替真噪声：

$$ \hat x_0 = \mathbb{E}[x_0\mid x_t]=\frac{x_t-\sqrt{1-\bar\alpha_t}\,\varepsilon_\theta}{\sqrt{\bar\alpha_t}} = \frac{x_t + (1-\bar\alpha_t)\,s(x_t,t)}{\sqrt{\bar\alpha_t}} $$

（两种写法等价，用了 \(s=-\varepsilon_\theta/\sqrt{1-\bar\alpha_t}\)。）这就是 Tweedie / Tweedie's formula。

4.2 DDPM 反向一步

已知 \(\hat x_0\) 后，用后验 \(q(x_{t-1}\mid x_t,\hat x_0)\)（一个高斯）采样下一步：

$$ x_{t-1}=\underbrace{\tilde\mu(x_t,\hat x_0)}_{\text{posterior mean}}+\sigma_t z,\quad z\sim\mathcal{N}(0,I) $$ diffusion.py · p_mean_variance (L81) + p_sample (L92) 
def p_mean_variance(self, x, t, classes=None):
    pred_noise = self.model(x, t, classes)
    # ① Tweedie：x_start = √(1/ᾱ_t)·x_t - √(1/ᾱ_t -1)·ε   （就是 x̂_0）
    x_start = sqrt_recip_alphas_cumprod[t]*x - sqrt_recipm1_alphas_cumprod[t]*pred_noise
    # ② DDPM 后验均值 = coef1·x̂_0 + coef2·x_t
    posterior_mean = posterior_mean_coef1[t]*x_start + posterior_mean_coef2[t]*x
    return posterior_mean, posterior_log_variance_clipped[t]

def p_sample(self, x, t, classes=None):
    mean, log_var = self.p_mean_variance(x, t, classes)
    noise = randn_like(x) if t>0 else 0.        # 最后一步 t=0 不再加噪
    return mean + (0.5*log_var).exp()*noise

@torch.no_grad()
def sample(self, classes=None, num_samples=None):

    for t in range(self.num_timesteps-1, -1, -1):     # T-1, T-2, ..., 0

到这里，一个完整的无条件扩散模型就齐了：第 1 节造数据、第 2 节加噪、第 3 节训 denoiser、第 4 节采样。main.py 里不加 --optimize 时跑的就是这套，生成的样本会落在数据流形上（off-support ratio ≈ 0）。

5引导采样：G / G_loss / 和 DPS 的关系

现在加入奖励。目标：让采样出来的 \(x_0\) 奖励高，同时留在数据流形上。做法是在第 4 节每一步的 score 上加一个引导项。

5.1 条件分数分解（所有 guidance 的出发点）

想从「高奖励的条件分布」\(p(x\mid y)\propto p(x)\,p(y\mid x)\) 采样，对 \(\log\) 求梯度：

$$ \underbrace{\nabla_{x_t}\log p(x_t\mid y)}_{\text{条件 score}}=\underbrace{\nabla_{x_t}\log p(x_t)}_{\text{无条件 score（denoiser）}}+\underbrace{\nabla_{x_t}\log p(y\mid x_t)}_{\text{guidance 项}} $$

难点在最后一项 \(p(y\mid x_t)\)：观测 \(y\) 是对干净 \(x_0\) 的，但我们手头是带噪的 \(x_t\)。三种方法的差别全在怎么近似这一项。

5.2 G（naive gradient，cond_score_version = v1）

直觉 最朴素：直接拿奖励梯度 \(g=\nabla f\) 当引导方向，往 \(g\) 推。它把「score 对 \(x_t\) 的依赖」近似成常数 \(g\)，没有考虑流形。
$$ G = s(x_t,t) + \beta_t\,(y - g^\top\hat x_0)\,g,\qquad \beta_t=\frac{\sqrt{\bar\alpha_t}}{\sigma^2 + (1-\bar\alpha_t)\|g\|^2} $$ diffusion.py · cond_score_v1 (L193) 
beta_t = alpha_t / (self.sigma**2 + h_t * norm(self.g)**2)     # 论文 Prop.3 的 β_t
x_0_hat = (x + h_t * uncond_score) / alpha_t                    # Tweedie
G = uncond_score + beta_t * (classes - sum(x_0_hat*self.g)) * self.g   # classes 就是 y
#                              ↑ (y - g^T x̂_0)              ↑ 方向固定为 g

5.3 G_loss（look-ahead loss，cond_score_version = v2）← 这就是 DPS

直觉 不直接用 \(g\)，而是定义一个「前瞻损失」\(\big(y-g^\top\hat x_0(x_t)\big)^2\)——衡量「如果现在收手去噪，得到的 \(\hat x_0\) 离目标多远」——然后对 \(x_t\) 求这个损失的梯度。关键：\(\hat x_0\) 是网络的输出，对它求梯度会穿过整个 denoiser 反向传播，于是引导方向被网络（也就是数据流形）「掰正」了。
$$ G_{\text{loss}} = s(x_t,t) - \beta_t\,\nabla_{x_t}\big(y-g^\top\hat x_0\big)^2,\qquad \beta_t=\frac{\bar\alpha_t/2}{\bar\alpha_t\sigma^2+(1-\bar\alpha_t)\big\|\nabla_{x_t}(g^\top\hat x_0)\big\|^2} $$ diffusion.py · cond_score_v2 (L155) 
with torch.enable_grad():
    x_in = x.detach().requires_grad_(True)
    pred_noise = self.model(x_in, t, classes=None)
    uncond_score = -pred_noise / noise_svd
    x_0_hat = (x_in + h_t * uncond_score) / alpha_t            # x̂_0(x_t)
    value = torch.sum(x_0_hat * self.g, dim=1, keepdim=True)   # g^T x̂_0  ← 线性观测 A(x̂_0)
    gradient = torch.autograd.grad(value.sum(), x_in)[0]       # ∇_{x_t}(g^T x̂_0)，★穿过网络★

beta_t = (alpha_t**2/2) / (alpha_t**2*self.sigma**2 + h_t*norm(gradient)**2)
gradient_2 = 2 * (value - classes) * gradient                  # ∇(g^T x̂_0 - y)^2
G = uncond_score - beta_t * gradient_2                          # 条件 score
论文对应 · v1 与 v2 的唯一本质区别 把 v2 的 gradient = ∇_{x_t}(g^T x̂_0) 近似成常数 \(g\)（即假装 \(\hat x_0\) 对 \(x_t\) 的雅可比是单位阵），v2 就退化成 v1。所以：
• v1：忽略 denoiser 雅可比 → 引导方向 = 原始梯度 \(g\) → 容易冲出流形（θ₂ 实验里 off-support ratio 飙升）。
• v2：保留 denoiser 雅可比 \(\partial\hat x_0/\partial x_t\) → 引导方向被投影回数据流形 → 奖励高且 ratio 低。
这个 denoiser 雅可比，就是论文反复强调的「流形信息」。

5.4 为什么说 v2 = DPS？

DPS（Chung et al., ICLR 2023）解逆问题 \(y=\mathcal{A}(x_0)+n,\ n\sim\mathcal{N}(0,\sigma^2)\)，它对 guidance 项的近似正是：

$$ \nabla_{x_t}\log p(y\mid x_t)\;\approx\;-\frac{1}{2\sigma^2}\nabla_{x_t}\big\|\,y-\mathcal{A}(\hat x_0(x_t))\,\big\|^2 $$

对比 5.3 的 \(G_{\text{loss}}\)：当观测算子取线性 \(\mathcal{A}(x)=g^\top x\) 时，两者逐项相同（论文的 \(\beta_t\) 把 \(1/2\sigma^2\) 这类系数吸收进去并按时间步自适应）。

这对你意味着什么 你不需要「另外实现一个 DPS」——v2 已经是一个 DPS 了，只不过观测算子被写死成线性的 \(g^\top x\)。你要做的事是把 value = sum(x_0_hat * g) 这一行换成 value = A(x_0_hat)，其中 \(\mathcal{A}\) 是你 metamaterial 的（可微）性质预测网络。见第 7、8 节。

5.5 引导如何接回采样链

算出条件 score \(G\) 后，转回「等效噪声」喂给第 4 节的 DDPM 后验，反向链其余部分原封不动：

diffusion.py · p_mean_variance 的 guided 重载 (L217)

6优化循环（Algorithm 1）

直觉 单次引导采样只是「朝目标 \(y\) 生成一批样本」。要做优化（不断把奖励顶上去），就把它套进一个外层循环：每轮用上一批样本的当前梯度重新设引导，目标值设成「比当前再高一点点」(\(\delta\)=interval)。逐轮逼近最优。
main.py · optimize (L304-340) —— 每一轮 
for i in range(args.opt_rounds):
    # 1. 在当前样本处查询奖励梯度 g_k = ∇f(z_k)（每个样本各一份）
    bs_gradient, bs_value = BatchGrad(query_samples, func)
    diffusion.update_grad(bs_gradient)                       # 存进 self.g

    # 2. 设目标值 y_k = δ + g_k^T z_k （在当前点线性外推，期望奖励涨 δ）
    guidance = interval + torch.sum(bs_gradient * query_samples, dim=1)

    # 3. 用引导扩散生成下一批样本（v1 或 v2）
    query_samples = diffusion.sample(num_samples=args.generate_bs, classes=guidance)

    # 4. 评估：奖励 + off-support ratio（看有没有偏离流形）
    reward = func(query_samples);  ratio = generator.off_support_ratio(query_samples)
    opt_logger.log(reward=reward.mean(), ratio=np.mean(ratio))

Algorithm 2 在此基础上加 --score_matching_finetune：用每轮新生成的高奖励样本对 denoiser 做几步 score-matching 微调（loss = diffusion(query_samples)，L320），让先验逐渐「跟上」优化方向。初学先忽略它，跑通 Alg.1 即可。

路 B：写一个「教科书版 DPS」函数（为搬去 metamaterial 铺路）

论文的 \(\beta_t\) 是自适应的、带很多项；标准 DPS 的写法更朴素，便于你之后改成非线性观测。核心就是把 5.3 的逻辑抽象成「任意可微观测算子 \(\mathcal{A}\) + 任意目标 \(y\)」。一个最小实现（可加进 diffusion.py 作为 cond_score_dps）：

建议新增 · diffusion.py · 教科书版 DPS（示意，A 为可微观测） 
def cond_score_dps(self, x, t, target, operator, zeta=1.0):
    """
    标准 DPS：guidance = - zeta/||y-A(x̂0)|| * ∇_{x_t} ||y - A(x̂0)||
    operator: 可微函数 A(x0_hat) -> 观测（线性时就是 g^T x；metamaterial 时是你的性质网络）
    target:   目标观测 y
    zeta:     步长超参（替代论文里复杂的 β_t；DPS 原文用 1/||y-A(x̂0)||）
    """
    h_t = extract(self.sqrt_one_minus_alphas_cumprod, t, (x.shape[0],1))**2
    alpha_t = extract(self.sqrt_alphas_cumprod, t, (x.shape[0],1))
    with torch.enable_grad():
        x_in = x.detach().requires_grad_(True)
        pred_noise = self.model(x_in, t, classes=None)
        uncond_score = -pred_noise / extract(self.sqrt_one_minus_alphas_cumprod, t, pred_noise.shape)
        x0_hat = (x_in + h_t * uncond_score) / alpha_t
        residual = target - operator(x0_hat)                 # y - A(x̂0)
        loss = torch.linalg.norm(residual)                   # ||·||（注意：范数，不是平方）
        grad = torch.autograd.grad(loss, x_in)[0]
    step = zeta / (loss.detach() + 1e-8)                     # DPS 原文的自适应步长
    G = uncond_score - step * grad
    return -1 * G * extract(self.sqrt_one_minus_alphas_cumprod, t, pred_noise.shape)  # 转回 noise
和 v2 的细微差别（别混淆） • v2 用的是 损失的平方 \((y-g^\top\hat x_0)^2\) 并带论文专属的自适应 \(\beta_t\)；标准 DPS 用 残差范数 \(\|y-\mathcal{A}(\hat x_0)\|\) 配步长 \(\zeta/\|\cdot\|\)。两者方向一致、缩放不同。
• 先用路 A（v2）确认能跑；路 B 只是为了让观测算子可插拔，方便第 8 节换成你的网络。两种都属于 DPS 家族。

8搬到 metamaterial 逆向设计

你的终极目标：设计一个 unit-cell 结构 \(x\)，使其物理性质（色散关系 / 带隙）命中目标 \(y^\star\)。这天然是个逆问题 → DPS 正合适。下面是「符号映射 + 改动清单」。

8.2 改动清单（基于路 B 的 cond_score_dps）

  1. 换网络维度：Unet1D → 2D U-Net（diffusers 的 UNet2DModel，或直接复用 image_generation/gradguided_sdpipeline.py 的结构）。\(\hat x_0\) 变成 2D 结构图。
  2. 换观测算子：把 operator 设成你的色散预测网络（必须可微、且 requires_grad=False 冻结其权重，只对输入 \(\hat x_0\) 求梯度）。这正是 image_generation/main.py 里 RCGDMScorer 扮演的角色——它就是个冻结的 ResNet 奖励模型，可微。
  3. 换目标：target = y\star（你的目标带隙）。损失用 \(\|y^\star-\mathcal{A}(\hat x_0)\|^2\)。
  4. 预训练扩散模型：先在你的结构数据集（datasetA?）上跑无条件预训练（第 2–4 节那套，2D 版）。这是前提，没有它 guidance 无从谈起。
  5. 调步长：metamaterial 的 \(\mathcal{A}\) 是强非线性，guidance 步长（\(\zeta\) 或论文 \(\beta_t\)）要重新调；太大→结构崩坏跑出流形，太小→性质不达标。这跟 image 实验里调 --guidance 是一回事。
直接参考 image_generation 而不是 simulations 你的结构是 2D 图像、观测算子是 NN——这跟 image_generation/gradguided_sdpipeline.py 结构完全一致（那里 \(\mathcal{A}=\) ResNet 奖励模型、\(x_0\) 经 VAE 解码成图像、对 latent 反传）。建议：把它当模板，把 RCGDMScorer 换成你的色散预测网络、把 target 换成目标带隙，就是你的 metamaterial DPS 雏形。simulations 用来吃透原理，image_generation 用来抄工程结构。
